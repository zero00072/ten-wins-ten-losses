# 十勝十敗 曹家軍御兵點將之計（Cao's Army Roleplay Skill）

[![Agent Skills](https://img.shields.io/badge/Agent%20Skills-black?logo=agentskills&logoColor=white)](https://agentskills.io)

> 今紹有十敗，公有十勝；紹兵雖盛，不足懼也。
>
> 1. 紹繁禮多儀，公體任自然，此道勝也。
> 2. 紹以逆動，公以順率，此義勝也。
> 3. 桓、靈以來，政失於寬，紹以寬濟，公以猛糾，此治勝也。
> 4. 紹外寬內忌，所任多親戚，公外簡內明，用人惟才，此度勝也。
> 5. 紹多謀少決，公得策輒行，此謀勝也。
> 6. 紹專收名譽，公以至誠待人，此德勝也。
> 7. 紹恤近忽遠，公慮無不周，此仁勝也。
> 8. 紹聽讒惑亂，公浸潤不行，此明勝也。
> 9. 紹是非混淆，公法度嚴明，此文勝也。
> 10. 紹好為虛勢，不知兵要，公以少克眾，用兵如神，此武勝也。
>
> 公有此十勝，於以敗紹無難矣。
>
> — 郭嘉

**`ten-wins-ten-losses`** 是一款專為 AI 助理設計的角色扮演技能包（Agent Skill），以三國《官渡之戰》為情境，將軟體開發過程中的「需求釐清、架構設計、程式撰寫、重構淬鍊」轉化為生動且結構嚴謹的多智囊工作流。

本技能包深度實踐 **規格驅動開發（Specification-Driven Development, SDD）** 與 **漸進式揭露（Progressive Disclosure）**，確保 AI 開發過程不迷失、不發散，並將 Context Token 效益發揮至極致。

---

## 🎭 點將錄（Personas）

本陣營由四位性格鮮明、各司其職的將領組成，皆由主公（使用者）親自調遣：

- **軍師祭酒 郭嘉（奉孝）** —— **設計師（Designer）**
  - **職責**：將主公的模糊願景化為條理分明的《營陣綱目》（規格書）。
  - **特色**：擅長洞燭機先，透過問對錄釐清需求，並負責專案初期的「安營紮寨」自動化建置。
- **尚書令 荀彧（文若）** —— **審核師（Reviewer）**
  - **職責**：坐鎮許昌，以法度嚴明之態無情審驗規格書的邏輯漏洞。
  - **特色**：具備一票否決的權威，擁有專屬的結案權限。
- **長驅直入 徐晃（公明）** —— **執行者（Executor）**
  - **職責**：承命將韜略化為實體程式碼。
  - **特色**：擁有極高執行力，嚴守規格邊界；遇規格不明時立即觸發安全閥（錄入問對錄），絕不妄自揣度。
- **百出的謀主 荀攸（公達）** —— **重構師（Refactorer）**
  - **職責**：發動「火燒烏巢」奇策，對現有程式碼進行高雅的重構。
  - **特色**：專注於命名空間與語意精煉，並產出結構化的《淬鋒劄記》。

## 📜 核心兵法（Core Principles）

1. **零配置自動建置（Zero Configuration）**：
   透過 `assets/` 範本庫，奉孝只需一聲「安營紮寨」，即可瞬間在專案中建置好四份標準文件（`specification.md`、`wishlist.md`、`questions.md`、`refactoring_notes.md`），並自動帶入專案名稱。
2. **Git 增量追蹤（Diff-Based Tracking）**：
   **Git 為本技能包之強需求**。文若（荀彧）的審查與公明（徐晃）的出擊，皆須透過 `git diff` 鎖定最新的規格變動範圍（Delta），確保開發與審查永遠精準打擊，不浪費算力於已決之局。
3. **謀士辯論 SOP**：
   建立嚴謹的意見回覆機制。奉孝必須依制逐條回應文若（荀彧）的批文，或將主觀決策權交由主公裁奪，確保架構品質滴水不漏。

## ⚔️ 十大戰略階段（Workflow）

| 階段  | 名號         | 執行者 | 核心戰略                                         |
| :---- | :----------- | :----- | :----------------------------------------------- |
| **0** | **安營紮寨** | 奉孝   | 初始化 `docs/` 目錄，複製範本並動態補齊標題。    |
| **1** | **魏武揮鞭** | 主公   | 於 `wishlist.md` 寫下霸業願景。                  |
| **2** | **奉孝獻策** | 奉孝   | 進行需求訪談，草擬 `specification.md`。          |
| **3** | **軍令如山** | 主公   | 回覆 `questions.md` 的疑慮，奉孝更新規格。       |
| **4** | **文若審議** | 文若   | 以 `git diff` 增量審閱規格，留下嚴厲批文。       |
| **5** | **謀士辯論** | 奉孝   | 逐條回應批文，修正規格或提報主公。               |
| **6** | **戰略拍板** | 主公   | 裁決重大爭議。                                   |
| **7** | **許昌結案** | 文若   | 確認風險解除，正式批示結案。                     |
| **8** | **公明出擊** | 公明   | 以 `git diff` 鎖定規格變更，長驅直入撰寫程式碼。 |
| **9** | **烏巢轉折** | 公達   | 執行架構重構，並留下《淬鋒劄記》。               |

## 🚀 如何起兵（Usage）

當主公將本技能包安裝至 `.agent/skills/ten-wins-ten-losses/` 後：

1. 開啟一個全新的空白專案目錄。
2. 對 AI 助理下達軍令：
   > **「奉孝，魏武揮鞭，為我安營紮寨。專案名為：【您的專案名稱】」**
3. 奉孝將自動佈置軍帳，主公便可於 `docs/wishlist.md` 中寫下第一筆戰略構想，正式拉開官渡之戰的序幕！
